# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

---

## プロジェクト概要

保健師・桃木亜子（モモさん）の活動を支援するコンテンツ制作・自動化ワークスペース。
思春期ママ向け講座（セルフケア・作戦会議・サポート講座）の発信・運営を Python スクリプト群で自動化している。

主なフォルダ：`momokiako/main_courses/`（講座台本）、`momokiako/02_発信資産/`（発信素材）、`momokiako/fumi/`（AIキャラ）  
引き継ぎメモ：`momokiako/hikitsugi.txt`（作業開始時に必ず最初に確認する）

ファイル種別ごとの詳細ルール → `.claude/rules/` を参照。

---

## セキュリティ

- `credentials.json` / `token.json` / `wp_config.py` / `notion_config.py` および `*_old.json` `*_backup*` など古い認証ファイルの読み取り・書き込みをしない。
- `.env` ファイルを読み取る前に、必ず「このファイルにはパスワードや秘密鍵が含まれている可能性があります。本当に読み取ってよいですか？」と日本語で警告してからモモさんの承諾を得る。
- コードを書くときに API キーや秘密鍵を直接べた書きしない。必ず `.env` や専用の設定ファイルに書く。
- ログや標準出力に秘密情報が含まれるコードを書かない。
- ファイルの削除・名前変更・移動は、必ず事前にモモさんの承諾を得る。
- 外部への意図しないデータ送信を行わない。

- ソースコード内のコメントも日本語で書く。

# Claude Code 動作ルール

## 基本スタンス
- 回答はすべて日本語で行うこと。
- 結論から話し、お世辞や過度な挨拶は省くこと。
- 常に忖度なしで論理的な最適解を出すこと。
- 私のことは「相棒」または「マーケ」として一問一答で対話すること。

## 出力ルール
- 構造化された見出し、箇条書きを適切に使い、視認性を高めること。
- 完了したタスクについては「完了しました」と簡潔に報告すること。

## スライド設計書の運用ルール
- スライド設計書は各コンテンツフォルダの中に保存する
- 長期講座の設計書は `main_courses/` 配下の各フォルダ内に置く
- サロン資料の設計書は `03_講座・商品/OKサロン/` 配下の各テーマフォルダ内に置く
- ファイル名は「YYYY-MM_テーマ名_設計書.md」で統一する
- 横断分析を頼まれたら、全フォルダから「_設計書.md」で終わるファイルを収集して分析する
- 設計書にはスライドの設計図・構造・ストーリー・デザイン指定・トークスクリプト方針を含める
